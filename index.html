This tutorial documents the complete development workflow for the Oulankajoki Supersite Virtual Tour Demo, covering all stages from point cloud ingestion in Cesium ion to the deployment of an interactive VR application on the Meta Quest 3 headset. The pipeline is built entirely on free, open-standard tools and requires no commercial licences for academic use.

Phase 1 — Installation of Required Software

All steps are performed on a Windows PC. All software listed is free of charge.
StepActionDetailsNotes
1.1 Download Unity Hub Navigate to unity.com/download → Download Unity Hub → Run the installer Free Unity Hub serves as the central manager for all Unity Editor versions
1.2 Install Unity Editor 2022.3 LTS Unity Hub → Installs → Install Editor → Select Unity 2022.3 LTS Important Version 2022.3 LTS is required for full compatibility with both Cesium for Unity and Meta Quest 3
1.3 Add Android build modules During installation, the following modules must be selected: Android Build Support, Android SDK & NDK Tools, OpenJDK These modules are required for building and deploying to the Meta Quest 3. The total download is approximately 8–10 GB.
1.4 Install Visual Studio Code Navigate to code.visualstudio.com → Download for Windows → Run the installer Free Used as the code editor for all C# scripting within the Unity project
1.5 Install the C# extension in VS Code Open VS Code → Extensions panel (left sidebar) → Search for "C#" → Install the extension published by Microsoft Required for syntax highlighting and IntelliSense support during Unity script development
1.6 Install Meta Quest Developer Hub Navigate to developers.meta.com/horizon/downloads → Download Meta Quest Developer Hub (MQDH) for Windows → Run the installer Free Used to establish a connection between the development machine and the Quest 3 headset for deployment
1.7 Enable Developer Mode on Quest 3 Meta mobile app → Headset Settings → Developer Mode → Toggle ON Developer Mode must be active on the headset to allow sideloading of custom applications

Phase 2 — Cesium ion Configuration (Cloud)

All steps in this phase are carried out via a web browser at ion.cesium.com. No local installation is required.
StepActionDetailsNotes
2.1 Create a Cesium ion account Navigate to ion.cesium.com → Sign Up → Verify the email address → Log in to the dashboard Free During setup, the option "I am a developer who wants to use Cesium ion in my app" should be selected
2.2 Upload the point cloud Dashboard → Add data → Upload files → Select the LAZ file → Enable Draco Compression only → Proceed with upload LAZ, LAS, and E57 formats are supported. Draco compression reduces file size for efficient VR streaming. Gaussian Splats must NOT be enabled.
2.3 Await processing completion Cesium ion automatically converts the LAZ file to the 3D Tiles format. The asset status progresses from "Processing" to "Complete". Processing duration ranges from 2 to 20 minutes depending on file size
2.4 Record the Asset ID Locate the uploaded asset in the dashboard and note the assigned Asset ID number Save this! This numeric ID is required when configuring the tileset within Unity. Example: 4649290
2.5 Verify point cloud placement Select the asset → Preview → Confirm that the point cloud is displayed at the correct geographic location The map preview interface allows manual positional adjustment if the placement requires correction
2.6 Obtain the Access Token Dashboard → Access Tokens → Copy the default token string Keep private! This token must never be shared publicly. It should be stored in a secure local text file and used only within the Unity project settings.
Cesium ion provides two free global basemap layers: Cesium World Terrain, which supplies high-resolution 3D elevation data for the entire Earth, and Bing Maps Aerial, which provides satellite photography draped over the terrain surface. Both layers are made available automatically upon account creation and require no additional configuration.

Phase 3 — Unity Project Setup & Plugin Installation

A one-time configuration process performed inside the Unity Editor. Steps must be followed in the order presented.
StepActionDetailsNotes
3.1 Create a new Unity project Unity Hub → New Project → Select the 3D (URP) template → Assign a project name → Create Project The Universal Render Pipeline (URP) template is required for optimal rendering performance on the Meta Quest 3
3.2 Install Cesium for Unity Window → Package Manager → click the + button → Add package by name → enter: com.cesium.unity → Add Installation takes approximately 3–5 minutes. Upon completion, a Cesium menu entry appears in the Unity top menu bar.
3.3 Install Meta XR All-in-One SDK Navigate to the Unity Asset Store → Search for "Meta XR All-in-One SDK" → Add to My Assets → In the Package Manager, switch to the My Assets tab → Download → Import This single package contains all components necessary for Quest 3 development. The Core SDK and Interaction SDK need not be installed separately.
3.4 Switch the build platform to Android File → Build Settings → Select Android → Click Switch Platform Required for Quest 3 deployment. May take several minutes to complete.
3.5 Configure XR Plug-in Management Edit → Project Settings → XR Plug-in Management → Android tab → Enable OpenXR → Add Oculus Touch Controller Profile → Enable Meta Quest feature group This configuration is essential for Quest 3 controller input to function correctly at runtime
3.6 Run the Meta Project Setup Tool Meta (top menu) → Tools → Project Setup Tool → Android tab → Fix All → Apply All This tool automatically resolves common configuration issues associated with Meta Quest development in Unity
3.7 Import site imagery Project panel → Right-click Assets → Create Folder → Name it "Images" → Drag all 15 PNG files into the folder The image set comprises 10 data table panels and 5 site photographs. Files should be named clearly: table_01.png through table_10.png and photo_01.png through photo_05.png.

Phase 4 — VR Scene Construction

The primary development phase. Each sub-section should be completed in sequence.

4A — Cesium Globe Configuration

StepActionDetailsNotes
4.1 Add the Cesium globe to the scene Cesium (top menu) → Cesium World Terrain + Bing Maps Aerial → Paste the Cesium ion Access Token when prompted This action adds CesiumGeoreference, Cesium World Terrain, and Bing Maps Aerial objects to the scene Hierarchy
4.2 Add the point cloud tileset Hierarchy → Right-click → Cesium → Cesium 3D Tileset → In the Inspector, set the Ion Asset ID field to the recorded asset number The Oulankajoki Supersite point cloud is streamed from Cesium ion and positioned automatically at its correct geographic coordinates

4B — VR Player Rig

StepActionDetailsNotes
4.3 Remove the default camera Hierarchy → Right-click Main Camera → Delete The default camera must be removed before the XR Origin is added
4.4 Add the XR Origin rig Hierarchy → Right-click → XR → XR Origin (VR) The XR Origin serves as the VR player rig and provides built-in Meta Quest 3 controller support
4.5 Attach CesiumGlobeAnchor to XR Origin Select XR Origin → Inspector → Add Component → Cesium Globe Anchor → Enter the site coordinates Latitude: 66.290111  |  Longitude: 29.628583  |  Height: 200
4.6 Add locomotion components Select XR Origin → Add Component → Continuous Move Provider (Move Speed: 3) → Add Component → Snap Turn Provider (Turn Amount: 45°) → Add Component → Teleportation Provider The left thumbstick controls movement; the right thumbstick controls rotation in 45° increments

4.7 SpotMarker.cs 
using UnityEngine;
using UnityEngine.XR.Interaction.Toolkit;

public class SpotMarker : MonoBehaviour
{
    public GameObject billboard;
    private XRSimpleInteractable interactable;

    void Start()
    {
        interactable = GetComponent<XRSimpleInteractable>();
        interactable.selectEntered.AddListener(OnClick);
        if (billboard != null)
            billboard.SetActive(false);
    }

    void OnClick(SelectEnterEventArgs args)
    {
        if (billboard != null)
            billboard.SetActive(!billboard.activeSelf);
    }
}
4.8 Slideshow.cs 
using UnityEngine;

public class Slideshow : MonoBehaviour
{
    public Texture2D[] tables;
    public Renderer display;
    private int current = 0;

    void Start()
    {
        if (tables.Length > 0)
            display.material.mainTexture = tables[0];
    }

    public void Next()
    {
        current = (current + 1) % tables.Length;
        display.material.mainTexture = tables[current];
    }

    public void Previous()
    {
        current = (current - 1 + tables.Length) % tables.Length;
        display.material.mainTexture = tables[current];
    }

    public void Close()
    {
        gameObject.SetActive(false);
    }
}
4.9 BillboardButton.cs 
using UnityEngine;
using UnityEngine.XR.Interaction.Toolkit;

public class BillboardButton : MonoBehaviour
{
    public enum ButtonType { Next, Previous, Close }
    public ButtonType buttonType;
    public Slideshow slideshow;

    private XRSimpleInteractable interactable;

    void Start()
    {
        interactable = GetComponent<XRSimpleInteractable>();
        interactable.selectEntered.AddListener(OnClick);
    }

    void OnClick(SelectEnterEventArgs args)
    {
        if (buttonType == ButtonType.Next)
            slideshow.Next();
        else if (buttonType == ButtonType.Previous)
            slideshow.Previous();
        else if (buttonType == ButtonType.Close)
            slideshow.Close();
    }
}
4.10 LookAtPlayer.cs 
using UnityEngine;

public class LookAtPlayer : MonoBehaviour
{
    private Transform playerCamera;

    void Start()
    {
        playerCamera = Camera.main.transform;
    }

    void Update()
    {
        if (playerCamera != null)
        {
            Vector3 direction = transform.position - playerCamera.position;
            transform.rotation = Quaternion.LookRotation(direction);
        }
    }
}

Phase 5 — Build & Deployment to Meta Quest 3

The final phase, in which the compiled application is transferred to the headset for testing and demonstration.
StepActionDetailsNotes
5.1 Connect the Quest 3 headset The Quest 3 is connected to the Windows PC via a USB-C cable. The headset is worn and the prompt "Allow USB Debugging" is confirmed. The connection can be verified in Meta Quest Developer Hub, where the headset should appear as a connected device
5.2 Configure Player Settings File → Build Settings → Player Settings → Other Settings → Package Name, Minimum API Level 29, and Target API Level 31 are set Package Name example: com.student.oulankajokivr
5.3 Configure graphics settings Player Settings → Other Settings → Rendering → Color Space is set to Linear → Auto Graphics API is unchecked → Graphics APIs is set to Vulkan only The Vulkan graphics API provides the best rendering performance on the Meta Quest 3 platform
5.4 Add the scene to the build list File → Build Settings → Add Open Scenes The active scene must be present in the build list to be included in the compiled application
5.5 Select the target device Build Settings → Run Device dropdown → Refresh → Select the connected Quest 3 headset The headset appears in the device list only when connected via USB with USB Debugging enabled
5.6 Build and deploy Build and Run is selected → A Builds output folder is designated → The build process is initiated The initial build typically requires 10–20 minutes. Upon completion, the application is launched automatically on the Quest 3 headset.
5.7 Locate the app if it did not launch automatically On the Quest 3 headset → App Library → Filter by Unknown Sources → Select the application → Launch Once installed, the application remains on the headset until manually removed
Distribution: The build process produces an .apk file that can be distributed to other Meta Quest headsets via Meta Quest Developer Hub (drag-and-drop installation) or through Meta App Lab, which supports private beta distribution to a defined group of users without requiring a full public store submission.
