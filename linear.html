Phase 1 — Software Installation
01
1.1 Download and install Unity Hub
Navigate to unity.com/download and download Unity Hub. Run the installer on Windows PC. Unity Hub manages all Unity Editor versions from a single interface.
unity.com/downloadWindows PCFree
02
1.2 Install Unity Editor 2022.3 LTS with Android modules
In Unity Hub go to Installs → Install Editor → select Unity 2022.3 LTS. During installation check Android Build Support, Android SDK and NDK Tools, and OpenJDK. These modules are required for Quest 3 deployment. Total download approximately 8-10 GB.
Unity 2022.3 LTSAndroid Build SupportAndroid SDK & NDKOpenJDK
Version 2022.3 LTS is mandatory for full compatibility with Cesium for Unity and Meta XR SDK.
03
1.3 Install Visual Studio Code and C# extension
Download from code.visualstudio.com. Open VS Code and install the C# extension published by Microsoft from the Extensions panel. This provides syntax highlighting and IntelliSense for all Unity scripting work.
code.visualstudio.comC# ExtensionFree
04
1.4 Install Git for Windows
Download from git-scm.com. Git is required for the Cesium for Unity plugin installation via GitHub releases. Without Git installed, Unity cannot process package URLs.
git-scm.comRequired for Cesium
05
1.5 Install Meta Quest Developer Hub
Download from developers.meta.com/horizon/downloads. Install Meta Quest Developer Hub for Windows. This tool establishes the USB connection between PC and Quest 3 for app deployment. Enable Developer Mode on the headset via the Meta Horizon smartphone app before connecting.
MQDHDeveloper ModeMeta Horizon AppUSB-C Cable
Developer Mode is activated via Meta Horizon mobile app → Devices → Quest 3 → Developer Mode → ON. Quest 3 software must be up to date.
Phase 2 — Cesium ion Cloud Setup
06
2.1 Create a free Cesium ion account
Navigate to ion.cesium.com and sign up. Verify email address. Select the developer option during setup. The free tier provides 5 GB asset storage, Cesium World Terrain, and Bing Maps Aerial — everything needed for this project at no cost.
ion.cesium.comFree TierDeveloper Account
07
2.2 Upload the Oulankajoki LAZ point cloud
Dashboard → Add data → Upload files → select the LAZ file (Oulankajoki_Jakalamutka_TM35FIN_N2000_allpoints_DSM25cm_color.laz, 27.7 MB). Enable Draco compression only. Do not enable Gaussian Splats. Cesium ion automatically converts LAZ to 3D Tiles format. Processing takes 2-5 minutes.
LAZ Format3D TilesDraco CompressionAsset ID: 4649290
The filename contains TM35FIN coordinate system data. Cesium ion reads these coordinates and reprojects automatically to WGS84.
08
2.3 Verify and adjust point cloud location on map
After processing completes, click on the asset and use the Preview map to confirm the point cloud appears at the correct location in Oulankajoki National Park, Finland. Manually adjust position if needed using the map interface. Site coordinates: 66°17'24.4"N 29°37'42.9"E.
66.290111°N29.628583°EOulankajoki Finland
09
2.4 Save the ion Access Token securely
Navigate to Access Tokens in the Cesium ion dashboard. Copy the default token string. Store it in a private text file on the local PC only. This token authorizes Unity to stream assets from Cesium ion. It must never be shared publicly or posted online.
Access TokenKeep PrivateRequired for Unity
Phase 3 — Unity Project Setup and Plugin Installation
10
3.1 Create a new 3D URP Unity project
Open Unity Hub → New Project → select the 3D (URP) template → assign a project name → Create Project. The Universal Render Pipeline template is required for optimal rendering performance on the Meta Quest 3 headset.
3D URP TemplateUniversal Render Pipeline
11
3.2 Install Cesium for Unity plugin via tarball
Navigate to github.com/CesiumGS/cesium-unity/releases in a browser. Download the latest .tgz release file. In Unity go to Window → Package Manager → click + → Add package from tarball → select the downloaded file. The Cesium menu appears in the Unity top menu bar when installation is complete.
GitHub Releases.tgz TarballPackage Manager3-5 min
12
3.3 Install Meta XR All-in-One SDK
Navigate to the Unity Asset Store and search for Meta XR All-in-One SDK. Click Add to My Assets. In Unity Package Manager switch to the My Assets tab, find the SDK, then click Download and Import. This single package contains all Quest 3 components including OVR Camera Rig, OVR Manager, controller input, and hand tracking.
Unity Asset StoreOVR Camera RigOVR ManagerController Input
Do not install Core SDK or Interaction SDK separately — the All-in-One SDK includes everything needed.
13
3.4 Switch platform to Android and configure XR
Go to File → Build Settings → select Android → Switch Platform. Then Edit → Project Settings → XR Plug-in Management → Android tab → check Oculus (uncheck OpenXR). Run Meta → Tools → Project Setup Tool → Android tab → Fix All → Apply All to automatically resolve common configuration issues.
Android PlatformOculus XR PluginProject Setup Tool
14
3.5 Import site images into Unity project
In the Project panel right-click Assets → Create Folder → name it Images. Drag all PNG image files into the folder. The set comprises 7 data table panel images (table_01 through table_07) and 5 site photography images (photo_01 through photo_05).

Phase 4 — VR Scene Construction
15
4.1 Add Cesium globe and point cloud tileset
Click Cesium in the top menu → Cesium World Terrain + Bing Maps Aerial. Paste the ion Access Token when prompted. This adds CesiumGeoreference, Cesium World Terrain (global 3D elevation), and Bing Maps Aerial (satellite photography) to the Hierarchy. Then right-click in Hierarchy → Cesium → Cesium 3D Tileset. In the Inspector set Ion Asset ID to 4649290 to stream the Oulankajoki point cloud.
CesiumGeoreferenceWorld TerrainBing Maps AerialAsset ID: 4649290
All scene objects with CesiumGlobeAnchor must be placed as children of CesiumGeoreference in the Hierarchy.
16
4.2 Set up OVR Camera Rig as VR player
Create an OVRCameraRig object under CesiumGeoreference. Add components: OVR Camera Rig, OVR Manager (set Target Device to Quest 3, Tracking Origin to Floor Level, Controller Driven Hand Poses to Controllers), Character Controller, Cesium Globe Anchor, and SetPlayerLocation script. Set player start coordinates to Latitude 66.290111, Longitude 29.628583, Height 200. Expand TrackingSpace and add OVR Controller Visual to LeftControllerAnchor and RightControllerAnchor.
OVRCameraRigOVR ManagerFloor LevelQuest 3
17
4.3 Write all required C# scripts in VS Code
Create a Scripts folder in Assets. Write and save eight scripts: SpotMarker (toggles billboard visibility on click), Slideshow (cycles through data table images with Next/Previous/Close), BillboardButton (handles button clicks), LookAtPlayer (billboard always faces the player camera), SetPlayerLocation (sets georeferenced coordinates via CesiumGlobeAnchor), VRMovement (thumbstick movement and A/B button flight), ControllerRay (draws ray lines and detects raycasted hits), TeleportSpot (gaze-based teleportation with 2-second hold).
SpotMarkerSlideshowBillboardButtonLookAtPlayerVRMovementControllerRayTeleportSpotSetPlayerLocation
18
4.4 Create DataMarker with slideshow billboard
Create a Sphere (Scale 0.5) named DataMarker under CesiumGeoreference. Attach Cesium Globe Anchor, SetPlayerLocation (coordinates 66.290111, 29.628583, Height 170), SpotMarker, Pointable Unity Event Wrapper, Ray Interactable, and OVR Raycaster. Create a child Quad named DataBillboard (Scale X9, Y6, Z1). Attach Slideshow script (assign all 7 table images), LookAtPlayer, and a black border frame Quad behind it. Create three child Cube buttons (NextButton green, PrevButton green, CloseButton red) each with BillboardButton script, OVR Raycaster, and TextMeshPro labels.
DataMarker SphereDataBillboard Quad7 Table SlideshowNext/Prev/CloseBlack Frame
19
4.5 Create 5 photo markers with image billboards
For each of the 5 site photos create a Sphere (Scale 0.5) named PhotoMarker01 through PhotoMarker05 under CesiumGeoreference. Assign unique georeferenced coordinates to each using SetPlayerLocation. Each marker has a child Quad billboard (Scale X18, Y12, Z1) with a photo material (PNG assigned to Albedo), LookAtPlayer script, and a black frame border. All billboards are set inactive by default and toggled visible when the ray trigger hits the parent sphere marker.
5 Photo MarkersPhoto BillboardsAlbedo TextureBlack FrameLookAtPlayer
Billboard Scale X18, Y12 is 3x larger than original to ensure readability in VR at typical viewing distances.
20
4.6 Add VR locomotion — movement and flight
Attach VRMovement script to OVRCameraRig. Left thumbstick controls forward/backward/left/right movement. Right thumbstick controls rotation. A button on right controller flies player upward for bird's eye view. B button descends back to ground level. Move Speed set to 50, Rotate Speed set to 90 for comfortable VR navigation across the geospatial terrain.
Left Stick — MoveRight Stick — RotateA — Fly UpB — Fly DownSpeed: 50
21
4.7 Add controller ray interactors
Attach ControllerRay script to OVRCameraRig. Two white LineRenderer rays are drawn from both left and right controllers using OVRInput position and rotation data. Right index trigger fires a Physics.Raycast that detects SpotMarker, BillboardButton, and TeleportSpot components on hit objects. Left index trigger also fires a raycast for marker interaction. Far clipping plane set to 500000 for full globe visibility.
LineRendererPhysics.RaycastBoth ControllersWhite RayIndex Trigger
22
4.8 Create 3 teleport hotspots
Create three yellow Cylinder objects (Scale X4, Y0.05, Z4) named TeleportSpot01, TeleportSpot02, TeleportSpot03 under CesiumGeoreference. Each has a SetPlayerLocation script at Height 160 with coordinates spread across the site. TeleportSpot script uses gaze-based detection — when the controller ray points at a cylinder for 2 seconds the cylinder color transitions from yellow to green and the player is automatically teleported to that location.
3 HotspotsYellow Cylinders2-Second GazeHeight: 160Auto Teleport

5.2 Configure Player Settings for Android
File → Build Settings → Player Settings → Other Settings. Set Package Name to com.student.oulankajokivr. Set Minimum API Level to Android 10 (API 29). Set Target API Level to Android 12 (API 31). Set Scripting Backend to IL2CPP. Set Target Architecture to ARM64. Set Graphics API to Vulkan only. Set Color Space to Linear.

5.3 Build and Run APK to Quest 3
File → Build Settings → click Add Open Scenes → select Quest 3 from the Run Device dropdown (click Refresh if not visible) → click Build and Run → create a Builds folder → save as OulankajokiVR.apk. First build takes 15-30 minutes. Subsequent builds take 3-5 minutes. The compiled APK is approximately 123 MB. App launches automatically on the headset when build is complete.

Final deliverable

Oulankajoki Supersite Virtual Tour — Complete VR Application

A fully standalone, georeferenced, interactive VR application running on Meta Quest 3. The point cloud and satellite basemap of the Oulankajoki Supersite stream live from Cesium ion cloud. The player navigates freely on foot and from the air, clicks sphere markers to open data panels and site photos, and teleports between key locations using gaze-based hotspots. The entire experience runs without any PC connection — requiring only the headset and a WiFi connection.
